# User Stories — 팀 일감 관리 앱

## 1. 핵심 시나리오

**페르소나:** 박지호 (일반 직원, 개발자) — 자신에게 할당된 일감을 매일 처리하는 팀원

---

### 하루 흐름 (시간 순)

| 시각 | 행동 | 앱에서 하는 일 |
|------|------|----------------|
| **09:00** 출근 직후 | 오늘 할 일 파악 | 담당자 필터에서 내 이름("박지호") 선택 → 할당된 일감 목록 확인 |
| **09:05** | 우선순위 확인 | 우선순위 "높음" 필터 적용 → 가장 급한 일감 1개 선택 |
| **09:10** | 작업 시작 선언 | 해당 카드 메뉴 → "진행 중"으로 상태 변경 |
| **11:30** | 오전 작업 완료 | 카드 메뉴 → "완료" 처리 / 취소선 + 흐림 처리로 완료 확인 |
| **11:35** | 다음 일감 선택 | 필터 유지 상태에서 "할 일" 컬럼의 다음 카드 확인 → 진행 중으로 변경 |
| **13:00** 점심 후 복귀 | 현황 재확인 | 보드 전체 훑기 — 내 일감 중 마감일 초과(빨간 날짜) 있는지 점검 |
| **15:00** | 일감 내용 수정 필요 | 카드 메뉴 → "수정" → 설명 업데이트 후 저장 |
| **16:30** | 오후 작업 완료 | 진행 중 카드 → "완료" 처리 |
| **17:50** 퇴근 전 | 내일 준비 | 남은 "할 일" 카드 마감일 확인 → 내일 처리할 순서 머릿속으로 정리 후 탭 닫기 |

---

### 핵심 기능 매핑

| 시나리오 단계 | 사용 기능 |
|--------------|-----------|
| 내 일감만 보기 | 담당자 필터 |
| 급한 것 먼저 보기 | 우선순위 필터 |
| 작업 시작·완료 표시 | 상태 변경 (할 일 → 진행 중 → 완료) |
| 내용 보완 | 일감 수정 |
| 마감 위험 감지 | 마감일 초과 시 빨간 날짜 표시 |

## 2. 결정 사항

1. **담당자도 일감의 상태를 자유롭게 변경할 수 있는가, 아니면 팀장만 변경할 수 있는가?**
   → 현재 MVP는 로그인이 없으므로 누구나 변경 가능. 추후 권한 분리가 필요한지 결정 필요.

2. **한 일감에 담당자를 여러 명 지정할 수 있는가, 아니면 1인만 가능한가?**
   → 현재 구조는 담당자 필드가 문자열 1개. 복수 담당자 지원 여부에 따라 데이터 모델이 달라짐.

3. **"진행 중" 상태의 일감이 동시에 여러 개 존재할 수 있는가, 아니면 1인 1개로 제한하는가?**
   → WIP 제한 없이 허용할 경우 병목 파악이 어려워질 수 있음.

4. **완료 처리된 일감을 삭제할 수 있는가, 아니면 이력 보존을 위해 삭제를 막아야 하는가?**
   → 현재는 삭제 허용. 완료 일감을 감사·회고 목적으로 보존해야 한다면 소프트 딜리트 또는 보관 기능이 필요.

5. **마감일 초과 알림은 앱 화면 내 표시(빨간 날짜)로 충분한가, 아니면 이메일·슬랙 등 외부 알림이 필요한가?**
   → 외부 알림은 1일 내 구현 범위를 벗어나므로 MVP에서 제외할지 명시적으로 결정 필요.
