# 요건 정의 — 팀 일감 관리 앱 (MVP)

> 근거 문서: `docs/personas.md`, `docs/user-stories.md`
> MVP 헌장: 단일 테이블 · 기능 5개 이하 · 1일 안에 코드로 옮길 수 있는 분량

---

## 1. 기능 요건

| ID | 이름 | 설명 | 우선순위 |
|----|------|------|----------|
| F-01 | 일감 CRUD | 제목·설명·담당자·마감일·우선순위를 입력해 일감을 생성·수정·삭제한다. 담당자는 문자열 1인으로 제한한다. | 필수 |
| F-02 | 상태 변경 | 일감의 상태를 할 일 → 진행 중 → 완료 세 단계로 전환한다. 권한 제한 없이 누구나 변경 가능하다. | 필수 |
| F-03 | 칸반 보드 뷰 | 상태별 3개 컬럼(할 일·진행 중·완료)으로 전체 일감을 한눈에 표시한다. | 필수 |
| F-04 | 필터링 | 담당자 이름 또는 우선순위(높음·보통·낮음)로 일감 목록을 즉시 좁혀 볼 수 있다. | 필수 |
| F-05 | 마감일 초과 표시 | 오늘 날짜를 지난 미완료 일감의 마감일을 빨간색으로 강조해 위험을 시각적으로 알린다. | 필수 |

## 2. 비기능 요건

### 성능

| 항목 | 기준 |
|------|------|
| 초기 페이지 로드 | Lighthouse Performance 점수 ≥ 80 (데스크톱 기준) |
| 필터 반응 속도 | 사용자 입력 후 화면 갱신 ≤ 100 ms (일감 100개 기준) |

### 보안

| 항목 | 기준 |
|------|------|
| 데이터 저장 범위 | 모든 데이터는 `localStorage`에만 저장, 서버 전송 없음 (MVP 기간 동안 외부 유출 경로 0건) |
| XSS 방지 | 사용자 입력을 DOM에 삽입 시 React의 기본 이스케이프 처리를 사용, `dangerouslySetInnerHTML` 미사용 |

### 가용성

| 항목 | 기준 |
|------|------|
| 오프라인 동작 | 네트워크 단절 상태에서도 `localStorage` 데이터로 읽기·쓰기 정상 동작 |
| 브라우저 호환 | Chrome · Edge · Safari 최신 버전 3종에서 주요 기능 100 % 동작 |

### 접근성

| 항목 | 기준 |
|------|------|
| 키보드 탐색 | 모든 인터랙티브 요소(버튼·입력·드롭다운)가 Tab 키만으로 접근·실행 가능 |
| 색상 대비 | WCAG 2.1 AA 기준 — 본문 텍스트 대비율 ≥ 4.5 : 1, 마감 경고 빨간색 포함 |
